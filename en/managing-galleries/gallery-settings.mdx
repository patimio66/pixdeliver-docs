## Overview

After creating a gallery, you can edit its settings at any time: change the name, slug (URL), description, password, and other parameters. All changes are immediate and visible to clients as soon as you save.

---

## Accessing gallery settings

**Instructions:**

1. Log in to PixDeliver
2. Go to **Galleries**
3. Find the gallery in the list and open it
4. Open the **Settings** tab
5. The gallery settings panel will open

## Editing basic information

### Changing the gallery name

**Instructions:**

1. In the **Basic** section, find the **Gallery name** field
2. Enter a new name (up to 100 characters)
3. Click **Save changes**

**Result:** The new name will be shown to clients on the gallery page and in your management dashboard.

<Info>
  The gallery name is visible to clients. It should be descriptive and professional.
</Info>

### Changing the slug (URL)

**Instructions:**

1. In the **Basic** section, find the **Slug** field
2. Enter a new slug (for example, "wesele-kasia-i-mariusz-2024")
   - Only lowercase letters, digits, and hyphens
   - At least 3 characters
   - Must be unique within your account
3. The system checks availability in real time
4. Click **Save changes**

**Result:** The gallery's URL will change. The old links will stop working.

<Warning>
  **Warning!** After you change the slug:

  - The old link stops working immediately
  - You must send clients the new link
  - Clients using the old link will see a 404 error
  - **<u>Only change the slug before sharing the gallery with clients!</u>**
</Warning>

### Editing the description

**Instructions:**

1. In the **Basic** section, find the **Description** field
2. Enter or update the gallery description
3. You can use formatted text and emoji
4. Click **Save changes**

**Result:** The description will be shown to clients below the gallery name.

<Tip>
  Next to the description field you'll find a **Generate with AI** button — it generates 3 suggested descriptions based on the gallery name and a short brief. If a description already exists, the **Replace** button swaps out the current text.
</Tip>

<Tip>
  **Ideas for description elements:**

  - "Download your favorite photos and share them with friends!"
  - "Thank you for a wonderful day! ❤️"
  - "Photos available to download until 12/31/2024"
</Tip>

## Managing the password

### Adding a password to a gallery

If the gallery was created without a password, you can add one at any time.

**Instructions:**

1. Open the gallery you want
1. In the **Settings** tab, enter a password
4. Save your changes

**Result:** From now on, clients will need to enter the password before accessing the gallery.

### Changing the password

**Instructions:**

1. In the **Security** section, find the current password (it's visible)
2. Enter a new password in the **Password** field
3. Confirm the new password
4. Save your changes

**Result:** The old password stops working. Clients must use the new password.

<Warning>
  If you change the password after sharing the gallery, you must let your clients know the new password. Otherwise, they'll lose access to the gallery.
</Warning>

### Removing the password

If you want to make the gallery publicly accessible without a password:

**Instructions:**

1. In the **Settings** section, clear the **Password protection** field
2. Save your changes

**Result:** The gallery becomes publicly accessible. Anyone with the link can view it.

## Delivery and sales

In the **Delivery and sales** section, you decide whether the gallery is free or paid:

- **Free** — clients download the originals right away
- **Paid — purchase of the entire gallery** — clients buy the whole gallery
- **Paid — purchase of individual photos** — clients buy the photos they choose

In the paid modes, you'll set the pricing here (**base price**, **price per photo**, **photos included**) and the **watermark text**.

<Warning>
  The delivery mode is **locked** while there's an active purchase request in the gallery. First settle or cancel the request in the **Purchase requests** tab.
</Warning>

<Info>
  Complete guides: [Selling galleries](/en/delivery-and-sales/selling-galleries) and [Purchase requests and settlement](/en/delivery-and-sales/requests-and-settlement).
</Info>

## Gallery URL

Each gallery has two possible URLs - but you can only use one type per account:

### 1. Default base address (root domain)

```
https://pixdeliver.com/g/[slug]
```

Example: `https://pixdeliver.com/g/wesele-jan-i-ania`

### 2. Subdomain address (if set)

```
https://[your-subdomain].pixdeliver.com/[slug]
```

Example: `https://jan-kowalski.pixdeliver.com/wesele-jan-i-ania`

<Tip>
  **A subdomain is more professional:**

  - It includes your name or company name
  - It builds brand recognition
  - It looks more personal
</Tip>

### Copying the link

**You can do this in a few ways:**

- In the gallery settings, find the **Public address** section and copy it
- Click the **Copy public link** button on any subpage of the gallery

**Result:** The link will be copied to your clipboard and you can send it to clients.

## Status and expiration

### Possible gallery statuses

- **Unpaid** - The gallery is not active and not publicly available
- **Active** - The gallery is paid for and available to clients
- **Expired** - The gallery has reached the end of its validity (after 1 year)

### Expiration date

For active galleries, you'll see the expiration date on the gallery list page:

**Example:**

```
Created: 06/15/2025
Expires: 06/15/2026
```

<Info>
  The system sends automatic notifications:

  - **30 days before expiration** - from this point you can renew the gallery for a year
  - **14 days before expiration** - a reminder
  - **7 days before expiration** - a reminder
</Info>

### Renewing a gallery

**Instructions:**

1. Open the gallery you want - you'll see an expiration notice
2. The plan stays the same as the one used when the gallery was created
3. Go to payment
4. Once paid, the gallery will be active for another year

**Result:** The expiration date will be extended by an additional 1 year.

## Deleting a gallery

<Danger>
  Before deleting a gallery, make sure you have backups of all the photos!
</Danger>

### Permanently deleting a gallery

If you no longer need a gallery:

**Instructions:**

1. Scroll to the bottom of the gallery settings
2. Find the **Danger zone** section
3. Click the **Delete gallery** button
4. Read the warning
5. Enter the gallery name to confirm
6. Click **Permanently delete gallery**

**Result:** The gallery and all its photos will be permanently deleted.

<Warning>
  **This action is irreversible!**

  - All photos from the selected gallery will be deleted
  - The gallery link will stop working
  - Statistics will be lost
  - Photo selections will be deleted
  - **There will be no way to recover the data**
</Warning>

## Exporting data

### Downloading all photos

Before deleting a gallery, or for archiving, you can download all the photos:

**Instructions:**

1. Open the gallery you want
1. Click the **Download** button in the gallery menu

**Result:** You'll download a ZIP file with all the photos from the gallery.

## Tips and best practices

✅ **Recommended:**

- Set a descriptive gallery name
- Add a description with instructions for clients
- Use a subdomain for a professional image
- Don't change the slug after sharing the gallery
- Regularly check your galleries' expiration dates
- Download a backup before deleting a gallery
- Set a password for sensitive sessions
- Renew galleries in advance (don't wait until the last day)

❌ **Avoid:**

- Changing the slug after sending the link to clients
- Deleting a gallery without downloading a backup
- Using the same password for every gallery
- Leaving a gallery unnamed or named "Test"

## Troubleshooting

**Problem:** I can't change the slug - it's taken. **Solution:**

- The slug must be unique within your account
- Check whether you already have a gallery with the same slug
- Add numbers, a date, or an extra description (for example, `-2024`, `-wesele`)

**Problem:** Clients can't open the gallery after I changed the slug. **Solution:**

- Send them the new link with the updated slug
- The old link no longer works and there's no redirect
- In the future, <u>don't change</u> the slug after sharing the gallery

**Problem:** The password doesn't work for clients. **Solution:**

- Check whether the password is entered correctly (it's case sensitive)
- Make sure the password doesn't have a space at the beginning or end
- View the current password in the gallery settings
- Send the password to clients again (copy it directly from the settings)

**Problem:** The gallery expired, but I don't want to delete it. **Solution:**

- Renew the gallery before 14 days pass after expiration
- After 14 days, the gallery may be deleted automatically
- Renewing it will restore the gallery's availability for another year

**Problem:** I don't see the "Save changes" button. **Solution:**

- Scroll to the end of the form - the button is at the bottom
- Refresh the page (Ctrl+F5)
- Check whether you've filled in all the required fields
